# Автомойка (ProjectAvtoMoika)

## 1. Краткое описание

**ProjectAvtoMoika** — настольное Windows Forms приложение для управления клиентами и услугами автомойки.  
Позволяет добавлять новых клиентов, просматривать административную панель, выполнять расчёты.  
Данные хранятся в локальной базе данных MySQL (работает через MAMP).

## 2. Инструкция по сборке и запуску

### 2.1. Требования
- **ОС:** Windows 7 / 10 / 11
- **Среда разработки:** Visual Studio 2019 / 2022 (или любая, поддерживающая .NET Framework 4.7.2)
- **.NET Framework:** 4.7.2 (указан в `App.config`)
- **MySQL Server:** Рекомендуется использовать MAMP (порт 3306) или любой другой MySQL 5.7+
- **NuGet пакеты:** Восстанавливаются автоматически при сборке (пакет `MySql.Data` указан в `packages.config`)

### 2.2. Подготовка базы данных
1. Запустите MAMP и убедитесь, что MySQL работает на порту **3306**.
2. Создайте новую базу данных с именем `avtomoika` (например, через phpMyAdmin).
3. Импортируйте структуру базы из файла `DataBase/avtomoika.sql`:
   - В phpMyAdmin выберите БД `avtomoika` → вкладка «Импорт» → выберите файл → «Вперёд».
   - Или через командную строку:
     ```bash
     mysql -u root -p avtomoika < DataBase/avtomoika.sql
    2.3. Настройка подключения
В корневой папке проекта скопируйте файл App.config.sample и переименуйте копию в App.config.

Откройте App.config и при необходимости измените параметры подключения:

xml
<connectionStrings>
  <add name="DefaultConnection"
       connectionString="server=localhost;port=3306;user id=root;password=ваш_пароль;database=avtomoika"
       providerName="MySql.Data.MySqlClient" />
</connectionStrings>
Важно: Файл App.config добавлен в .gitignore и не попадает в репозиторий.

2.4. Сборка и запуск
Откройте решение ProjectAvtoMoika.sln в Visual Studio.

Дождитесь автоматического восстановления NuGet-пакетов.

Убедитесь, что в проекте добавлена ссылка на System.Configuration (если нет — добавьте через «Ссылки» → «Добавить ссылку»).

Нажмите F5 для запуска приложения.

4. Инструкция по использованию
После запуска приложения доступны следующие формы:

Главное окно (вероятно, стартовое меню выбора действий).

Добавление клиента (AddClient):
Введите имя клиента, номер автомобиля и контактный телефон. При дублировании данных приложение предупредит, но продолжит работу.

Панель администратора (AdminPanel):
Просмотр и управление списком клиентов и операций.

Расчёт стоимости (FormCacl):
Калькулятор для подсчёта стоимости услуг.

Бронирование (FormRezerv):
Форма для создания/просмотра бронирований.

Список операций (list_operations):
История оказанных услуг.

Все взаимодействия с базой данных происходят через класс DB, который читает строку подключения из App.config.

6.2. Известные особенности
Порт MySQL по умолчанию 3306. Если ваш MAMP использует порт 8889, отредактируйте строку подключения в App.config соответствующим образом.

Приложение разработано для однопользовательского режима (локальная БД). Для многопользовательской работы потребуется доработка и центральный сервер.
